## 新增功能

### 1. iOS 支持 (dev.5 新增)
- 新增 iOS 构建 job (ARM64 Device / ARM64 Simulator)
- 产物：`tthsd_ios_arm64_device.dylib`、`tthsd_ios_arm64_simulator.dylib`
- 最低 iOS 版本：13.0 (修复 zstd 链接问题)

### 2. 自动重试机制 (dev.5 新增)
- 支持指数退避算法
- 默认：初始延迟 1s，最大延迟 30s，最大重试 3 次
- 可通过 ABI `set_retry_config()` 运行时配置

### 3. 速度限制 (dev.5 新增)
- 支持通过 ABI `set_speed_limit(speed_bps)` 设置下载速度限制
- 参数 speed_bps 为 bytes/s，0 表示不限速

### 4. 代理支持 (dev.5 新增)
- 支持 HTTP/HTTPS/SOCKS5 代理
- 可通过 ABI `set_proxy(proxy_url)` 运行时设置
- 使用 NULL/None 禁用代理

### 5. 性能统计 (dev.5 新增)
- 新增 ABI `get_performance_stats()` 返回 JSON 格式统计信息
- 包含：总下载量、当前速度、平均速度、峰值速度、分块下载数、失败/重试数、运行时间
- 需配合 `free_string()` 释放内存

### 6. 自定义 Headers (dev.5 新增)
- 全局 Headers：所有任务共享，通过 `headersJson` 参数设置
- 任务级 Headers：每个任务独立，通过任务数据的 `headers` 字段设置
- 任务级会覆盖全局同名 Header

## API 变更

新增 ABI 函数：

| 函数 | 描述 |
|------|------|
| `set_speed_limit(id, speed_bps)` | 设置速度限制 |
| `set_proxy(id, proxy_url)` | 设置代理服务器 |
| `set_retry_config(id, max_retries, delay_ms, max_delay_ms)` | 配置重试参数 |
| `get_performance_stats(id)` | 获取性能统计 |
| `free_string(ptr)` | 释放 get_performance_stats 返回的内存 |

`start_download` / `get_downloader` 新增参数 `headersJson`

## 构建配置变更

### UPX 版本更新
- 旧版本：apt-get 安装的系统版本
- 新版本：使用 `crazy-max/ghaction-upx@v3` 明确指定 v5.1.1
- Linux 和 HarmonyOS 构建均已更新

### Android 库名变更
- 旧名称：`libTTHSD.so`
- 新名称：`libtthsd.so`

### 新增配置文件
- `.cargo/config-macos-ios.toml` - macOS/iOS 构建配置

## 错误处理改进

- 改进的错误信息格式，更友好的错误描述
- 支持常见 HTTP 错误码（502/503/504/404/403）的中文提示
- 网络错误（连接拒绝、超时、无法路由等）的清晰描述
- 文件系统错误（磁盘空间不足、权限拒绝等）的明确提示
